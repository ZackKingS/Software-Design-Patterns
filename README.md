# 软件设计模式：老鸟带小白的奇妙之旅

> 一本用「对话」讲透设计模式的轻松读物。资深架构师 **王哥** 带着刚入职的 **小白**，从一个个真实的「代码翻车现场」出发，把 SOLID 原则和 GoF 全部 23 种设计模式，讲成一个个鲜活的生活场景。

## 这本书有什么不一样

- **故事连贯**：全书是一条线索串起来的。每一章开头，都在回答上一章王哥留下的思考题；每一章结尾，又抛出一个新难题，引出下一章的模式。从第 1 章读到第 18 章，像追一部连续剧。
- **痛点驱动**：不堆砌定义。每个模式都从「小白写出了一坨没人敢碰的烂代码」讲起，先体会痛，再请出模式，最后对比重构前后的差别。
- **代码说话**：所有示例用 **Java**，配 ❌ 烂代码 / ✅ 好代码 对照，并附 **Mermaid UML 类图**帮助理解结构。
- **拒绝炫技**：反复强调「**简单优先、不要为了用模式而用模式**」——知道一个模式存在，和该不该用它，是两回事。

## 人物设定

- **王哥**：10 年+ 经验的后端架构师，说话一针见血，爱用生活比喻解释复杂设计，常年手握一杯冰美式。
- **小白**：入职不到一年的程序员，热情但常写出「意大利面条」式代码，总被不断变化的需求折磨。

## 每章结构

1. **小剧场**：小白遇到一个棘手的需求变更，代码改得痛不欲生。
2. **核心概念**：王哥点出问题根源，用比喻引出对应模式。
3. **代码重构**：一步步把烂代码重构成好代码（❌ / ✅ 对照）。
4. **模式精讲**：UML 类图、适用场景、优缺点、与相似模式的对比。
5. **课后总结**：小白的笔记，外加王哥留下的思考题（承接下一章）。

## 如何阅读

直接从 [第 1 章](chapter_1.md) 顺序往下读即可——由于每章首尾相扣，**强烈建议按顺序阅读**，不建议跳读。

支持 Mermaid 的 Markdown 阅读器（如 GitHub、Typora、VS Code）可直接渲染书中的 UML 类图。

- 准备面试、需要快速回顾 23 种模式的同学。
